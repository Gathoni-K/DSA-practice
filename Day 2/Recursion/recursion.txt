6.
-How to actually use recursion in coding.
-Steps:
1. Understand the problem.
2. Identify the subproblem.
3.Trust / Faith.
//trust that the recursive call ill correctly solve a smaller version of my problem
//assume that if your function works for simpler cases it will work for more complex ones.
4. Link 1 and 2.
5. Base condition.

__Print a sequence that looks like this: (n, n-1....1, 1,2...n)
__Our subproblem:  print 2,1,1,2
__Trust: on setting our function correctly and determining the correct base case, recursion
saves one from mentally unrolling all the 'inner workings'.
__Link: in the output ensure subproblem is combined in the problem 
__Base condition: ensure it is well written to avoid an infinite loop.

Pseudocode:
function seq(n) {
    if (n=0) return;
//our base case
    print n;
    seq(n-1);
//our inductive step
    print n;
}
seq(3);

4.
Psuedocode for factorial using iteration.
fact = 1;
for 1 = 5 to 1;
fact = fact * 1;
return fact;

-Things done recursively can also be done iteratively.
-Iteration does not use space on the recursion call stack.
-Better space complexity is obtained when using iteration.
-Recursion has both an ascending(calling) and descending(returning) phase.
-Iteration only has the ascending phase.
-Solutions written recursively are more readable and easily written.
